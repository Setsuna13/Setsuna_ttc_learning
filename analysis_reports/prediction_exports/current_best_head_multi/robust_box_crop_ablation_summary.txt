Conclusion:
Eval-only robust crop is not suitable for the current trained checkpoint. It can rescue some extreme annotation/crop outliers, but it changes the crop distribution for many samples that the current model already predicts well, especially in 0~3s and 3~6s, causing a net metric regression. Keep the feature disabled by default. If this direction is pursued, it should be used during training as a consistent preprocessing/augmentation policy, or converted into a conservative anomaly flag/sample weight rather than direct eval-time crop replacement.
